## 4. 继续定位：不是“没上传”，而是“优化链路失败”

### 已验证通过的部分

- `/api/admin/uploads` 上传成功（200）
- `/api/admin/products` 保存成功（200）
- `/api/products` 返回了 `coverImageUrl`（含 R2 公网 URL）
- 说明：
  - 不是“上传失败”
  - 不是“前台没拿到图片 URL”

### 失败点

- Next.js 日志出现：
  - `upstream image ... resolved to private ip ["198.18.x.x"]`
- 这说明：
  - `next/image` 服务端优化器在抓远程图时被安全策略拦截
  - 非图片资源本身不可用

## 5. 稳定优先修复：产品相关远程图改 `unoptimized`

- 对产品相关前台页面的远程图添加 `unoptimized`
- 结果：
  - 本地恢复正常显示
  - 线上也恢复正常显示
- 结论：
  - 当前环境下，稳定可用方案是“远程图绕过 Next 优化器，浏览器直连加载”
