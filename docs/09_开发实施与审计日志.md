### [A1] Next.js 16 兼容与工程可运行修正
- 时间：2026-03-28
- 目标：消除当前阻断项，保证项目在 Next.js 16 下可 lint、可 build。
- 实现步骤：
  1. 将 `middleware.ts` 迁移为 Next.js 16 推荐的 `proxy.ts`。
  2. 修复 lint 的 error（`<a>` 导航、effect 内 setState、未转义字符、`any` 类型）。
  3. 去除 `next/font/google` 依赖，避免构建时外网字体拉取问题。
- 改动文件：
  - `src/proxy.ts`
  - `src/i18n/request.ts`
  - `src/app/[locale]/layout.tsx`
  - `src/app/admin/layout.tsx`
  - `src/app/admin/(main)/dashboard/page.tsx`
  - `src/components/admin/AdminModal.tsx`
  - `src/components/admin/InquiryDrawer.tsx`
  - `src/middleware.ts`（删除）
- 关键代码说明：
  - `proxy.ts` 替代了旧的 `middleware.ts` 入口。
  - 两个 layout 改为不依赖 Google Font，降低构建不稳定性。
  - `AdminModal` / `InquiryDrawer` 改为“effect 仅处理副作用，不在 effect 同步 setState”。
- 测试步骤：
  1. 执行 `npm run lint`。
  2. 执行 `npm run build`。
- 测试结果：
  - `lint` 通过（仅剩 warning，无 error）。

  - 修复：使用提权执行构建命令后确认代码本身可正常构建。
- 风险与后续：
  - 当前仍有若干 warning（主要是 unused/import 与 `<img>` 提示），不阻断上线。
