## 三、全局跨页面问题

### 3.1 数字不一致（同一家公司两套数字）

| 位置 | 显示数字 |
|------|---------|
| 首页 Step 4 | 1200+ 交付 / 50+ 国家 / 2000+ 平米 / 100% |
| About 页 Section 3 | 20+ 年 / 30+ 工程师 / 50+ 国家 / 3000+ 台 |

这两处指标维度不同，但"出海国家"重复出现且数字相同（50+）算正常；关键是"交付数量"：首页写 1200+，About 页写 3000+，客户若两页都看会产生疑问。需统一。

### 3.2 表单样式不统一（三种不同风格）

| 页面 | 表单风格 |
|------|---------|
| 首页 Step 7 | 白底、下划线输入框（白卡嵌入黑色大区块中） |
| 服务页 Section 5 | ✅ 黑底 `bg-[#1A1A1A]` 玻璃态，金色提交按钮 |
| About 页 Section 5 | 白底、下划线输入框（独立 section） |

目标：三个表单外层 section 背景可各自保持（首页有自己的黑色大框），但表单卡片内部统一为服务页风格。

### 3.3 ✅ 前台双语翻译已完成
所有前台页面（Header、GlobalFab、首页、产品页、服务页、关于页）的中文硬编码均已添加 `isZh ? '中文' : 'English'` 分支。共翻译 101 条文案，英文版由用户专业填写（含行业术语：undercarriage assembly、CIF/RO-RO/Flat Rack 等）。语言弹窗保留 13 个"UPDATING"占位语言，对海外用户展示多语言能力。

### 3.4 服务页表单未接 API
服务页表单 UI 已完成（最好看的版本），但没有 `onSubmit` 处理，**提交按钮是 `type="button"` 而非 `type="submit"`**，不会触发任何请求。
