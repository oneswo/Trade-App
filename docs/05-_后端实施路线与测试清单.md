## 1. 当前结论（代码审计结果）

基于当前仓库状态，结论如下：

1. 询价主链路已闭环：前台多入口提交、后台读取、标记已读、基础防刷均已可用（当前为 Mock Repo）。
2. 后台登录与路由保护已可用：已支持登录 Cookie 与 `/admin/*` 保护。
3. 后台产品管理已闭环：列表、新建、编辑、删除、图片/视频上传均已落地（当前上传写本地 `public/uploads`）。
4. 已预留后续切换点：数据层与存储层已抽象，部署前可切换到 Supabase + R2。
5. 前台产品页已支持“假数据/真实 API”切换结构，当前默认继续用假数据展示。
6. 当前剩余关键工作是部署联调：Supabase 真库、R2 真存储、Vercel 环境变量与上线回归。

结论：当前版本已满足“本地业务功能闭环”，离上线只差部署前真实服务接入与回归。

## 5. 上线验收 10 条用例（必须全绿）

1. 未登录访问 `/admin/inquiries` 会被重定向到 `/admin/login`。  
2. 后台错误密码登录失败，页面有错误提示。  
3. 后台正确密码登录成功，进入仪表盘。  
4. 首页询价提交成功，Supabase `inquiries` 新增记录。  
5. 联系页询价提交成功，Supabase `inquiries` 新增记录。  
6. 产品详情页询价提交成功，Supabase `inquiries` 新增记录。  
7. 后台询盘列表能看到最新询盘在顶部。  
8. 询盘详情抽屉能显示姓名/联系方式/留言内容。  
9. 标记已读后刷新页面状态不丢失。  
10. `npm run lint` 与 `npm run build` 均通过。

## 6. 我们接下来的执行方式

按阶段 A → B → C（本地纯 Mock 闭环）→ D（部署前一次性接真实 API）串行推进。  
每做完一个子功能（A1/B1/C1…）立即执行对应测试并记录结果，再进入下一项。  
这样可以在开发阶段完全不依赖外部环境，最后只集中做一次真实联调，减少打断和返工。
